size_t  size

Function lib_imt_copy performs internal memory transfers between any two memory locations local to the C66x CorePac: L1P, L1D, and L2. This function uses IDMA channel 1 to perform the transfer.

Parameters
[in]srcsource address of transfer
[in]dstdestination address of transfer
[in]sizesize of transfer in minimum access units
Remarks
This function waits for the pending transfer (if any) to become active. It is a blocking call.
There is no need to allocate a channel before using this function.
Both source and destination addresses increment linearly throughout the transfer.
No DSP interrupt is generated after the transfer. Function lib_imt_wait can be called to wait for transfer completion before data in destination memory is used.
During any conflicts between CPU and DMA accesses, transfer started by this function has the lowest priority.
